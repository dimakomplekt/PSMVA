# Функционал программы по основным частям

## Запуститься и выдать меню

* Дать возможность начать работу с программой или выйти (мб через крест)
* GUI в общем стиле

## Принять файлы высокоскоростной видеосъмки процесса плазменного напыления и характеристики этих файлов (кол-во кадров в секунду, время экспозиции)

* Кнопка выбора файла / файлов - поддержка 2 вариантов выбора - по одному (система + 1) или сразу несколько
* Дефолтный обозреватель WINDOWS + передача выбора из обозревателя
* GUI в общем стиле
* Справка по месту или интуитивно понятный интерфейс выбора или всё вместе
* Переход к следующему этапу или выход (мб через крест)

## Дать пользователю настроить отфильтрованные OpenCV-маски (фейдерами регулировки различных параметров масок) для обнаружения различных составляющих процесса плазменного напыления (струя плазмы, мелкодисперсные частицы) + дать возможность сохранить настройки в какой-то файлик и взять настройки из какого-то файлика

* Селектор выбора выбранных на этапе 1 видео для настройки
* Кнопки выбора этапа настройки (типо 1 (дефолтная позиция), 2, 3, 4, Параметры видео, Общие) с перекраской из красного в зеленый (или иные) после завершения настройки + общий этап (расписан ниже)
* Свои фейдеры настройки маски под каждый этап
* Окно с базовым видео + окно с превьюером видео с работой текущей маски режима 
* На этапе настройки параметров видео - возможность ввода (или парсинга / автопарсинга через выбор файлов (если будет время)) основных параметров видео - количества кадров в секунду, количества кадров, времени экспозиции
* На общем этапе настройки - возможность сохранения или загрузки пресета параметров масок
* GUI в общем стиле
* Справка по месту или интуитивно понятный интерфейс выбора или всё вместе
* Переход к следующему этапу (только при заполнении всех данных - блокировка при отсуствии даты) или выход (мб через крест)

## По полученным маскам составляющих произвести анализ файлов высокоскоростной видеосъмки процесса плазменного напыления

* При отстройке - работа алгоритмов с визуализацией
* В рабочей версии - просто фоновые вычисления требуемых параметров по логике, согласно алгоритму
* Передача результатов формирователю отчёта
* При ошибке обработки - сигнализация, предоставление лога, переход на 2 этап
* GUI в общем стиле - просто какой-то прогресс-бар по каждому видео (можно, допустим не полностью и подряд, а подряд поэтапно) + статус бар по этапам обработки каждого выбранного видео - можно в форме таблички

#### Форма статуса расчёта:


| Статус                                        | Видео 1 | Видео 2 |
|-----------------------------------------------|---------|---------|
| Этап 1                                        |    x    |    +    |
| Этап 2                                        |    -    |    -    |
| Этап 3                                        |    -    |    -    |
| Прогресс                                      |   15%   |   45%   |
___
