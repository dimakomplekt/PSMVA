# **Приложение для анализа видео процесса плазменного напыления**

Я создаю программу, необходимую для подтверждения или опровержения влияния работы созданного моей рабочей группой регулятора степени турбулизации потока на турбулизацию потока плазмы в плазмотроне (соотв. для подтверждения или опровержения концепции влияния изменения турбулизации потока плазмы на общую эффективность процесса плазменного напыления).

Программа должна:

* принять файлы высокоскоростной видеосъмки процесса плазменного напыления и характеристики этих файлов (кол-во кадров в секунду, время экспозиции)
* дать пользователю настроить отфильтрованные OpenCV-маски (фейдерами регулировки различных параметров масок) для обнаружения различных составляющих процесса плазменного напыления (струя плазмы, мелкодисперсные частицы) + дать возможность сохранить настройки в какой-то файлик и взять настройки из какого-то файлика
* по полученным маскам составляющих произвести анализ файлов высокоскоростной видеосъмки процесса плазменного напыления
* сформировать отчет с обозначением численных характеристик параметров динамики различных составляющих процесса плазменного напыления, позволяющих сделать выводы об изменении турбулизации потока
 


В рамках обработки предполагается определять такие параметры, как:

* Средняя интенсивность излучения при различных режимах работы – показатель, отражающий усредненную яркость зоны плазменного напыления - позволяет оценить мощность излучения потока по значениям для референсного светильника с известными параметрами.
* Средняя амплитуда поперечного размера струи по оси Y – величина, характеризующая динамические процессы в плазменном потоке – позволяет сопоставить данные об устойчивости струи при различных режимах модуляции.
* Средняя скорость движения частиц в основном направлении (по трекам и значению экспозиции камеры) – параметр, вычисляемый при анализе смещения контрастных объектов между последовательностью кадров, используется для анализа кинематических свойств струи и эффективности модуляции.
* Процент частиц с выраженным отклонением от основного направления движения – интегральная характеристика уровня турбулизации потока, отражающая степень «разброса» траекторий относительно основного направления.
* Средний угол отклонения частиц от основного направления – показатель среднего уровня неустойчивости и возмущенности струи.
* Максимальный угол отклонения частиц от основного направления – критерий экстремальных флуктуаций и глобальной устойчивости структуры движения частиц.

Отчет по анализу каждого видео должен представлять из себя таблицу вида:

| Параметр                                      | Режим 1 | Режим 2 |   ...   | Режим N |
|-----------------------------------------------|---------|---------|---------|---------|
| Средняя интенсивность излучения               |         |         |         |         |
| Средняя амплитуда струи                       |         |         |         |         |
| Средний угол направления частиц               |         |         |         |         |
| Средний угол отклонения частиц                |         |         |         |         |
| Скорость частиц в основном направлении        |         |         |         |         |
| Скорость отклоненных частиц                   |         |         |         |         |
| Процент частиц с выраженным отклонением       |         |         |         |         |
| Число Рейнольдса                              |         |         |         |         |

## **Основной алгоритм программы**

Для поиска данных значений будет использован следующий алгоритм:

1) Произвести улучшение качества видеоматериала с использованием алгоритмов нейросетевого апскейлинга.
2) Произвести поиск значений интенсивности светового потока в процессах с регулируемыми модуляциями, используя grayscale-отображение кадров видео и функцию измерения среднего значения цвета всех пикселей в кадре, опираясь на предварительно полученное референсное значение интенсивности излучения типового светильника (зафиксированного при одинаковых настройках камеры).
3) Произвести, используя трекбары контроля переменной x (номер пикселя по горизонтали) для нескольких вертикальных прямых ручное разделение кадра на участки: зоны сопла плазмотрона, зоны начального участка струи, зоны переходного участка струи, зоны основного участка струи.
4) Для каждого участка вручную произвести при помощи трекбаров контроля параметров hsv-маски (Hue, Saturation, Value для «cv::inRange»), трекбаров контроля параметров фильтра размытия по Гауссу (для «cv::GaussianBlur»), трекбаров контроля максимальной длины и площади контуров для результирующей функции «cv::findContours», а также - для маски треков частиц – трекбаров параметров «cv::Canny» и «cv::dilate» настройку фильтрованных модульных масок для поиска контуров объектов в кадре: маски сопла (только в зоне сопла), маски струи (везде, кроме зоны сопла) и маски частиц (везде, кроме зоны сопла).
5) По полученной маске сопла определить, используя две крайние точки правой грани сопла, его центр и уравнение прямой, совпадающей с осью сопла (перпендикуляр к правой грани). Зная размеры плазмотрона, определить геометрические характеристики кадра (реальный размер 1 пикселя в профиле рабочей зоны).
6) По геометрическим параметрам кадра и полученным маскам струи для трёх участков найти значение средней амплитуды струи на данных участках.
7) По геометрическим параметрам кадра и полученным маскам частиц для трёх участков покадрово определить, начальные (A) и конечные (B) точки треков частиц, сформировать массивы данных, включающие в себя: координаты точек A и B; угол направления движения частицы относительно оси сопла - α, найденный, через уравнение для поиска угла между прямой оси сопла и прямой, проведенной, через точки A и B; скорость частицы на данном участке – v, найденную, через уравнение для поиска длины вектора по записанному времени экспозиции кадра.
8) По заполненным массивам данных покадрово определить средний угол направления движения всех частиц в кадре – β и усреднить, как среднее арифметическое по количеству кадров. Покадрово определить средний угол значительного отклонения частиц – γ, как среднее арифметическое для всех частиц с углом α в пределах [ β-180°,β-5°]∪[ β+5°,β+180°] и усреднить, как среднее арифметическое по количеству кадров.
9) Покадрово определить и усреднить, как среднее арифметическое по количеству кадров средние скорости движения частиц, двигающихся в основном направлении и средние скорости движения частиц, двигающихся с значительным отклонением, относительно основного.
10) Покадрово найти отношение количества частиц с углом α в пределах [ β-180°,β-5°]∪[ β+5°,β+180°] к общему количеству частиц в кадре. Определить (как среднее арифметическое по количеству кадров) средний процент частиц с выраженным отклонением от основного направления для всех кадров.
11) Рассчитать примерное число Рейнольдса по известному значению массового расхода плазмообразующего газа.

## **Принципы построения программы**

#### **Простой вариант**

Сделать всё одеялом функциями OpenCV - быстро, более-менее понятно, но не прикольно - оставить на крайний случай.

#### **Непростой вариант**

Взять мой движок на базе собственной state-машины и SDL3 из проекта (https://github.com/dimakomplekt/miyoo_square), подшаманить движок с игры на программу, придумать нормальный GUI, реализовать (украсть отсюда - https://stackoverflow.com/questions/22702630/converting-cvmat-to-sdl-texture) хороший транслятор типов - из cv::Mat в SDL_Texture, делать через OpenCV лишь  логику обработки кадров, всё остальное строить на базе движка и SDL3.

Не быстро, не очень понятно, но очень прикольно - потраить до 1 апреля, при провале забрать всю логику обработки кадров и долбануть первый вариантик.
